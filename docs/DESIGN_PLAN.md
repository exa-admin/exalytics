# Design-First Plan

## Purpose

Design the product before writing application code so the frontend, backend, database, and CMS all support the same user experience.

## Current Phase 1 Pack

Detailed Phase 1 files live in `docs/phase-1-design/`.

Start with `docs/phase-1-design/README.md`, then use the product definition, sitemap, screen specs, wireframe notes, design system direction, and Figma task list to create the Figma file.

## Phase 1: Product Definition

Outputs:

- Product name or working title.
- Target users.
- Main navigation.
- First topic definition.
- Success criteria for MVP.

Decisions:

- Which topic categories exist first.
- Which pages are public.
- Which pages require admin login.

## Phase 2: Information Architecture

Outputs:

- Sitemap.
- Topic page structure.
- Admin section structure.
- Content model for notes and methodology.

Core public pages:

- Homepage.
- Topics listing.
- Topic dashboard.
- Analysis note.
- Collaborate.
- Collaborate topic proposal.
- Login.
- Sign up.
- Subscription.

Core admin pages:

- Topic manager.
- Indicator manager.
- Observation import or upload.
- Dashboard widget editor.
- Notes editor.
- AI/CMS review queue.
- Refresh history.

## Phase 3: Wireframes

Create low-fidelity Figma wireframes for:

- Homepage desktop.
- Homepage mobile.
- Topics listing.
- Balance of Payments India dashboard.
- Notes/article page.
- Admin notes editor.

Keep the wireframes focused on layout and user flow, not colors.

## Phase 4: Visual Design System

Define:

- Typography.
- Color palette.
- Spacing scale.
- Chart colors.
- Buttons.
- Filters.
- Cards or panels.
- Data tables.
- Forms.
- Empty states.
- Loading states.
- Error states.

The style should feel analytical, calm, and credible.

## Phase 5: High-Fidelity Figma Mockups

Create polished designs for:

- Homepage.
- Topic dashboard.
- Topics listing.
- Notes page.
- Admin editor.
- Collaborate proposal page.
- Admin AI/CMS review page.
- Login page.
- Sign-up page.
- Subscription and billing page.

Use realistic example data for Balance of Payments of India.

## Phase 6: Interactive Prototype

Prototype:

- Search to topic page.
- Topic card to dashboard.
- Date range filter.
- Goods/services toggle.
- Open note from dashboard.
- Admin edit note flow.
- Collaborate proposal submission flow.
- AI draft review to publish flow.
- Login and sign-up flow.
- Subscription upgrade flow.

## Phase 7: Design Review Checklist

Before frontend development:

- Mobile and desktop layouts exist.
- Navigation is clear.
- All dashboard widgets have data requirements.
- All filters are defined.
- CMS fields are known.
- Empty/loading/error states are designed.
- Colors work for multiple chart series.
- Topic pages can support future dashboards.
