# Phase 1: Website And Product Design

## Goal

Complete the product and website design before writing application code. Phase 1 should make the app clear enough that Phase 2 can build the Blazor frontend with mock data.

## Design Principle

Design this as an economic analytics portal. The first screen should help users find a topic, see what changed recently, and open a dashboard. It should not feel like a generic blog or marketing landing page.

## Phase 1 Deliverables

- Product definition.
- Sitemap.
- User flows.
- Screen inventory.
- Low-fidelity wireframe specs.
- Visual design system.
- Figma task list.
- Design review checklist.

## Primary Screens

- Homepage.
- Topics listing.
- Topic dashboard.
- Analysis note page.
- Collaborate page.
- Admin login.
- Admin dashboard.
- Admin topic manager.
- Admin notes editor.
- Admin data upload or refresh page.

## First Topic For Mockups

Use Balance of Payments of India in all high-fidelity examples.

Show example dashboard data for:

- Current account balance.
- Goods exports.
- Goods imports.
- Services exports.
- Services imports.
- Net goods balance.
- Net services balance.
- Current account as percent of GDP.

## Exit Condition

Phase 1 is complete when the Figma file has desktop and mobile designs for the public experience, core admin screens, reusable components, and enough notes for a developer to build Phase 2 without guessing the layout.
